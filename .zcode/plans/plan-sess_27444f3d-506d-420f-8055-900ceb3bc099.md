# Axon-UI Redesign · Phase 2 — "Midnight Glass"

## Context
Phase 1 (CSS consolidation) is complete: `style.css` is 2,587 lines with 1 `!important`, dead components are deleted, `LoginPage` is migrated, build passes, all 28 tests green. This plan implements the chosen **Midnight Glass** aesthetic direction by retuning the existing `:root` token system and adding a scoped frosted-glass treatment. No fonts change; no icons change; no structural template changes.

## Locked decisions
- **Direction:** Midnight Glass (deep cool surfaces, electric-blue accent, cyan focus glow, softer radii, frosted chrome + panels)
- **Fonts:** keep Space Grotesk / Instrument Sans / IBM Plex Mono (no new downloads, CSP `font-src 'self' data:` untouched)
- **Glass scope:** chrome bars + `.panel` + `.modal` only. Buttons, inputs, `.card`, chat bubbles, `.item`, `.list-row` stay solid for text crispness.

## Step 2 — Ambient body background (`style.css` body, ~lines 177–185)

Add a fixed cool glow layer so translucent surfaces have something to refract (on desktop the topbar is a flex sibling of the scroller, so this is what gives the glass real depth):
```css
body {
    background:
        radial-gradient(ellipse 60% 50% at 15% -5%, color-mix(in srgb, var(--accent) 12%, transparent), transparent 70%),
        radial-gradient(ellipse 50% 45% at 85% 0%, color-mix(in srgb, var(--accent-glow) 9%, transparent), transparent 70%),
        radial-gradient(ellipse 70% 50% at 50% 110%, color-mix(in srgb, var(--blue) 7%, transparent), transparent 70%),
        var(--bg);
    background-attachment: fixed;
}
```

## Step 3 — Glass on chrome bars
- `.shell-topbar` (lines 553–562): `background: var(--glass-bg)`, add `backdrop-filter: blur(var(--glass-blur))` + `-webkit-` prefix, keep hairline border.
- `.sidebar` (lines 278–291): `background: var(--glass-bg)` + blur (both prefixes). Drawer + collapse transitions unaffected.
- `.mobile-tabbar` (lines 2403–2419): already glassy (`blur(16px)`, `color-mix(...88%)`); realign to `var(--glass-bg)` / `var(--glass-blur)` / `var(--glass-border)` for consistency.

## Step 4 — Glass on `.panel` (lines 769–775)
`background: var(--glass-bg)`, `border-color: var(--glass-border)`, add a 1px inset top highlight via `box-shadow: inset 0 1px 0 rgba(255,255,255,0.04), var(--shadow-sm)`, plus `backdrop-filter: blur(var(--glass-blur))` (both prefixes). `.card`, `.item`, inputs, buttons unchanged (stay solid; they already read tokens so they auto-darken).

## Step 5 — `.modal` (lines 1338–1348) + `.modal-overlay` (lines 1322–1332)
- `.modal`: translucent tint `background: var(--glass-bg)`, `border-color: var(--glass-border)`, inset top highlight. **No** `backdrop-filter` on the modal itself — the overlay already blurs behind it, so a second blur would go muddy.
- `.modal-overlay`: bump `blur(6px)` → `blur(var(--glass-blur))` for consistency (both prefixes already present).

## Step 6 — Focus halo (lines 1254–1260, input/textarea/select:focus)
Add a faint cyan outer glow reinforcing the electric feel:
```css
box-shadow: 0 0 0 3px color-mix(in srgb, var(--accent) 16%, transparent),
            0 0 14px color-mix(in srgb, var(--accent-glow) 22%, transparent);
```

## Step 7 — Login page gradient (lines 2080–2083)
Retune the radial glows to the new accent/glow/blue palette over a deeper base, so the gate matches the shell.

## Step 8 — Theme color (`index.html` line 40 + `manifest.json` lines 9, 10)
`#16181d` → `#0d1017` (browser chrome + PWA splash match the new deep bg).

## Step 9 — Accessibility safety guard
Add near the motion-discipline block (lines 2575–2587):
```css
@media (prefers-reduced-transparency: reduce) {
    .shell-topbar, .sidebar, .mobile-tabbar, .panel, .modal {
        background: var(--bg-card);
        backdrop-filter: none;
        -webkit-backdrop-filter: none;
    }
}
```

## Step 10 — Preserve the instrument well (lines 2169–2194)
Leave `#page-workflows, .nd-overlay, .curl-modal-overlay` token override **exactly as-is**. It's already deeper (`--bg #0a0a0c`) and self-contained; the new app bg (`#0d1017`) brings the shell closer to it, reducing disconnect. The canvas keeps its own teal accent (`#56e6c8`) — a known, accepted pre-existing condition. (A canvas-accent retune can be a separate follow-up if desired.)

## Risk notes
- `backdrop-filter` perf on low-end devices → mitigated by the `prefers-reduced-transparency` guard and by scoping glass to chrome + panels (not everywhere).
- Glass translucency reads as subtle over a near-solid bg on desktop; the ambient body glow (Step 2) is what sells it.
- One commit at the end; branch off `main` first since this is the default branch.
