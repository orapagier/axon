# Axon-UI Redesign Plan: Two Phases

## Phase 1 — CSS Consolidation (the cleanup)

### 1.1 Rewrite `src/style.css` (5,236 → ~1,200 lines)

**Delete entirely** (~3,500 lines):
- "Modern dashboard refresh" layer (lines 1908–2730)
- "2026 minimalist refresh" layer (lines 2905–3870)
- "LEGACY SURFACE REMAP" layer (lines 4855–4985)

**Keep as-is** (~260 lines):
- "MOBILE NATIVE LAYER" (lines 4987–5236) — tab bar, safe areas, iOS zoom fix

**Merge + rewrite** (~900 lines):
- Base tokens (`:root`) — keep the GRAPHITE LAB palette, add missing tokens, clean up dead tokens (`--node-border-running`, `--node-border-pinned`, `--node-accent`)
- Component rules (`.card`, `.item`, `.chat-*`, etc.) — single definition each, no `!important`
- POLISHED BUTTON SYSTEM — strip all `!important`, keep the ~10 classes actually used in templates, delete ~24 orphaned btn classes
- GRAPHITE LAB v4 kit (`.page-wrap`, `.panel`, `.row-list`, `.list-row`, `.switch`, `.empty-state`, `.mono-chip`) — promote from "authoritative override" to primary definition
- Search field system — keep, clean up
- Chat markdown rendering — keep, one definition
- Preserve `#page-workflows, .nd-overlay, .curl-modal-overlay` instrument-well token override block exactly

**Outcome:** One coherent token-driven layer, zero `!important` wars, zero duplicated definitions.

### 1.2 Migrate LoginPage from legacy to token-driven

LoginPage is the only page still using `premium-card`, `premium-input`, `form-group-modern`:
- Replace `premium-card` → `panel` / custom `.login-card` using tokens
- Replace `premium-input` → standard `input` token styling (already handled globally)
- Replace `form-group-modern` → `.form-grid` / `.form-field` (pattern already in ModelsPage)

### 1.3 Delete dead code

- Delete 5 unused component files: `components/nodes/{JsNode,ShellNode,SynapseNode,TriggerNode}.vue` and `components/edges/ButtonEdge.vue`
- Delete `public/favicon1.png` (79KB orphan, not referenced in index.html)

### 1.4 Verify

- Run `npm run build` — confirms no broken references
- Run `npm test` — confirms 5 existing tests stay green
- Manual visual check of each page in browser

## Phase 2 — New Aesthetic Direction (via ui-ux-pro-max-skill)

### 2.1 Install + query the skill

- Install `ui-ux-pro-max-skill` (clone or npx)
- Query for dashboard/agent-panel specific recommendations:
  - Color palettes (dark-theme, data-dense)
  - Font pairings (display + body + mono for a dev-tools aesthetic)
  - UI style direction (refined graphite? soft-glass? neo-brutalist? warm-dark?)

### 2.2 Present options to user

- Show 2–3 aesthetic directions with sample token values
- User picks a direction (or stays with refined graphite)

### 2.3 Apply chosen direction

- Retune `:root` tokens (palette, radius scale, shadow intensity)
- If new fonts: download variable woff2 → `public/fonts/`, update `@font-face`, update `manifest.json` + `index.html` theme-color
- If icon style changes: update inline SVGs in `App.vue` NAV array
- Re-verify build + tests + manual visual check

## What stays untouched (by design)

- **Canvas system** (9 components, NodeDetails, WorkflowsPage scoped styles) — already self-contained dark literals; only the `#page-workflows` override block in global CSS is touched (preserved, not rewritten)
- **ChatPage scoped styles** (`conv-*`, voice controls, mobile bar) — fully token-driven already; no changes needed
- **5 vitest tests** — behavioral, not visual; unaffected by CSS
- **Build pipeline** — no changes to vite.config, prebuild script, or CSP
